# T2 — `archive-artefacts.py`: dir-aware + JSON-twin + `--slug` mode

**Specialist:** `backend-reviewer`, `security-reviewer` · **Batch 1** · parallel with T1

## Task
Extend the existing archiver so it (a) archives **directories** (brief dirs) not just `*.md` files, (b) collects a slug's viewer **JSON twin**, and (c) exposes a **`--slug <slug>`** targeted mode that archives one slug's whole scope in a single promote-then-move — the reusable primitive `reap.py` (T3) delegates to. Keep the daily session-start sweep and the existing `--file` mode behaviorally unchanged.

## Why
The script currently skips directories (`:288-289` `if not p.is_file() … continue`), so brief dirs (`tasks/<slug>/`) accumulate as orphans (2 on disk now). Viewer JSON twins (`artefacts/*/<slug>.json`) are never touched by the archiver at all. `reap.py` needs one canonical "promote learnings then move to archive/" — that logic already lives here and must not be duplicated.

## Scope
- IN: directory archival, JSON-twin collection, `--slug` mode, path-safety assertion, tests.
- OUT: no change to the daily-sweep mtime policy, `--file` semantics, or memory promotion format.

## Files in scope
- `scripts/archive-artefacts.py`
  - **Directory archival:** add an archive path for a directory target (brief dir `tasks/<slug>/`, feature tree already handled at `:170-194`). Reuse `archive_feature`'s rglob-promote-then-move approach for a generic dir. Destination `archive/<type>/YYYY-MM/<name>/` (bucket by mtime month, per `:144-149`).
  - **JSON twin:** given a slug + type, if `artefacts/<type>/<slug>.json` exists, move it alongside its archived artefact into the same `YYYY-MM` bucket.
  - **`--slug <slug>` mode:** new argparse path that resolves the slug's scope (`tasks/<slug>.md` + `tasks/<slug>/` + `specs/<slug>.md` + `specs/<slug>.draft.md` + `features/<slug>/` + matching `artefacts/*/<slug>.json`), promotes learnings from each (reuse the promote helper at `:125`), then moves each to archive. Returns a JSON summary on stdout: `{archived:[{path,dest}], promoted:[...]}`.
  - **Path safety:** validate slug `^[a-z0-9-]+$`; `Path.resolve()` every source/dest and assert it is under the passed `.hyperflow` root before any `os.replace`/move. Refuse otherwise.
- `tests/test_archive_artefacts.py` — extend.

## Acceptance criteria
1. A brief dir `tasks/<slug>/` is archived (moved, learnings promoted) — no longer skipped.
2. `artefacts/task/<slug>.json` is archived alongside `tasks/<slug>.md`.
3. `--slug <slug>` archives the full scope in one call and prints the JSON summary.
4. Slug failing `^[a-z0-9-]+$`, or any resolved path escaping the `.hyperflow` root, is refused with a non-zero exit.
5. The existing daily sweep (`main()` / `--file`) produces identical output to before on a fixture (regression).

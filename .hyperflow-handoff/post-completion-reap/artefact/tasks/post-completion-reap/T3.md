# T3 — `scripts/reap.py` core + tests

**Specialist:** `architect`, `backend-reviewer`, `security-reviewer` · **Batch 2** · depends T1, T2

## Task
Create the scope-aware reaper: given a slug, resolve its whole artefact scope, dispose each class per the policy, and emit a JSON report. This is the engine behind both the auto-hooks (T6) and `/hyperflow:reap` (T4).

## Why
No single pass GCs a task's whole scope today — deletion/archival is scattered across dispatch (ad-hoc delete), the daily sweep, and manual skills. Reap unifies them, archive-first and memory-preserving, with one report.

## Scope
- IN: `scripts/reap.py` + `tests/test_reap.py`.
- OUT: no skill edits (T4/T6); no config schema (T1); reuse T2 for archive+promote.

## Interface
`python3 scripts/reap.py <hf_root> --slug <slug> [--dry-run] [--force] [--json]`
- Reads `cleanup.*` via the same `load_cfg` used by `archive-artefacts.py`.
- Prints a JSON report to stdout (always); human summary to stderr unless `--json`.

## Behavior (disposition classes — see spec §1)
1. **Resolve scope + terminal state.** Detect flat task (`tasks/<slug>.md`), feature (`features/<slug>/feature.md`), or spec-only. Terminal = flat status `complete`/`completed` (tolerate the inconsistent `Status|in_progress` vs `State|complete` schemas — match case-insensitively on the value), or `feature.md` Status `completed`, or `--force`/manual. Non-terminal without `--force` → no-op with reason.
2. **Archive class (reversible):** delegate to `archive-artefacts.py --slug <slug>` (T2) → captures its JSON summary. Do NOT re-implement move/promote.
3. **Ephemeral class (hard-delete):** `usage/*.jsonl` with mtime older than `usageRetentionDays` (never the `.active-chain-id`/`.chain-base` pointers if a chain is active); truncate `.session-start.log` to the last `logMaxLines`; delete `background/bg-*.md` whose registry entry is terminal (`complete|error|stalled|cancelled`) and >7d (reuse the background prune rule); remove an **empty/settled** `commits-queue/` only (never one with an unflushed manifest).
4. **Memory class (preserve + optimize):** rebuild `index.md` (invoke `scripts/memory-index.py`); drop entries whose `Evidence:` source file no longer exists (reuse the cache orphaned-ref rule); flag/compact files over `memory.compactionThreshold`. **Never delete a durable entry.**
5. **Report:** aggregate `{slug, dryRun, archived:[], deleted:[], bytesFreed, memory:{indexRebuilt, orphansDropped, compacted:[]}, skipped:[{path,reason}]}`.

## Safety (non-negotiable)
- Slug validated `^[a-z0-9-]+$` before any FS op.
- Every target path `.resolve()`-d and asserted under `<hf_root>` (or the sibling `.hyperflow-handoff` only when explicitly passed) — refuse traversal.
- `--dry-run` (or `cleanup.dryRun`) performs ZERO mutation; report marks `dryRun:true` and lists the plan.
- Idempotent: a second reap of the same slug finds nothing → clean no-op report.
- Never touch `.version`, `.last-cleanup`, active queue, in-flight artefacts, or `.hyperflow-handoff` package dirs (handoff `complete` owns those).

## Acceptance criteria
1. Reaping a completed flat task moves task file + brief dir + JSON twin to `archive/`, leaves durable memory intact, deletes only stale ephemera, emits the report.
2. Non-terminal slug without `--force` → no-op, exit 0, report `skipped:[{reason:"non-terminal"}]`.
3. `--dry-run` mutates nothing (assert filesystem identical before/after).
4. Second reap → empty no-op.
5. Traversal slug refused, exit ≠ 0.

## Test cases (`tests/test_reap.py`)
- Completed-task fixture → assert moves, memory preserved, report shape.
- In-progress fixture → skipped.
- `--dry-run` on completed fixture → zero mutation, plan populated.
- Idempotency: reap twice → second is empty.
- Orphaned memory ref (Evidence points at a deleted file) → dropped; a valid durable entry → kept.
- `--slug ../evil` → refused.
- **E2E:** build a temp `.hyperflow/` fixture with a completed task + brief dir + twin + a durable learning + a stale `usage/*.jsonl` + a bloated `.session-start.log`; run reap; assert archive contents, memory intact, log truncated, usage gone, report accurate.

## Gotchas
- Promote-before-move ordering is inside T2 — don't duplicate it; just call and trust the summary.
- Do not delete `usage/*.jsonl` for a chain that is still active (respect `.active-chain-id`).
- `.session-start.log` truncation must keep the tail (most recent), not the head.
- stdlib only (match the rest of `scripts/`).

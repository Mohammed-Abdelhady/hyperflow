# T5 — Register the reap skill in manifests + README

**Specialist:** `devops-reviewer`, `docs` · **Batch 3** · depends T4

## Task
Make the new `reap` skill discoverable across the surfaces the other skills are registered in (Claude Code plugin + Codex plugin), and list it in the README skill table.

## Why
A `SKILL.md` alone is not invokable until it is registered in the plugin manifest(s) the same way `cache`/`background`/`flush` are. Without this, `/hyperflow:reap` won't route and Codex won't see it.

## Scope
- IN: manifest/registration edits + README skill row.
- OUT: no SKILL.md content changes (T4); no doctrine (T7).

## Files in scope
- Plugin registration under `.claude-plugin/` (the marketplace/plugin manifest that enumerates skills — locate how `cache`/`flush` are listed and add `reap` identically).
- `.codex-plugin/` equivalent registration.
- Any central skills index/features listing the repo maintains (grep for an existing skill slug like `flush` or `background` to find every file that must list `reap`).
- `README.md` — add a `reap` row to the skill table with trigger phrases: `/hyperflow:reap`, "clean up the finished task", "reap this task", "garbage collect".

## Acceptance criteria
1. `reap` appears wherever `cache`/`background`/`flush` appear — no surface missed.
2. Claude Code + Codex both resolve `/hyperflow:reap`.
3. README skill table lists reap with a one-line description + triggers.
4. No manifest fails its own schema/lint.
